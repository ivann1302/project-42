# Как на сайте собираются лиды и где может идентифицироваться трафик

Этот документ описывает текущую логику сбора заявок на сайте Project 42. Его можно использовать как контекст для дальнейшего анализа приватности, защиты персональных данных и минимизации идентификации трафика.

Важно: цель анализа - законная защита приватности пользователей, снижение лишнего трекинга, корректная работа с персональными данными и уменьшение утечек технических идентификаторов. Не цель - обход правил рекламных платформ, антифрод-систем, аналитики или сокрытие запрещенной активности.

## Краткая схема

Сайт сделан на Next.js/React. Лиды собираются двумя основными способами:

1. Через обычную форму заявки `ContactForm`.
2. Через CTA-квиз в блоке `Cta`.

Обе формы отправляют данные POST-запросом на endpoint:

```text
NEXT_PUBLIC_CONTACT_ENDPOINT или /scripts/api/send.php
```

Если переменная `NEXT_PUBLIC_CONTACT_ENDPOINT` не задана, используется локальный PHP-обработчик:

```text
public/scripts/api/send.php
```

После успешной отправки пользователь перенаправляется на страницу:

```text
/thank-you
```

Заявка доставляется в Telegram через Telegram Bot API.

## Обычная форма заявки

Компонент:

```text
src/features/ContactForm/ui/ContactForm.tsx
```

Схема валидации:

```text
src/features/ContactForm/model/schema.ts
```

Поля формы:

```text
name        - имя, обязательно, от 2 до 100 символов
phone       - телефон, необязательно, но нужен phone или email
email       - email, необязательно, но нужен phone или email
service     - тип проекта, необязательно
message     - сообщение, необязательно, максимум 1000 символов
_honeypot   - скрытое поле против простых ботов, должно быть пустым
_page       - путь текущей страницы, добавляется при отправке
```

Список вариантов услуги в форме:

```text
Лендинг
Корпоративный сайт
SEO-оптимизация
Поддержка и развитие
Настройка и запуск рекламы
Другое
```

Клиентская валидация:

```text
React Hook Form + Zod
```

Клиент отправляет JSON:

```json
{
  "name": "Имя пользователя",
  "phone": "+7...",
  "email": "example@company.ru",
  "service": "Лендинг",
  "message": "Текст сообщения",
  "_honeypot": "",
  "_page": "/tekuschaya-stranica"
}
```

После успешной отправки обычная форма дополнительно отправляет цель в Яндекс.Метрику:

```text
contact_form_submit
```

## CTA-квиз

Компонент:

```text
src/widgets/Cta/ui/Cta.tsx
```

Квиз состоит из 4 шагов:

```text
1. Сфера деятельности
2. Готовность контента для сайта
3. Срочность
4. Контакт и удобный способ связи
```

Варианты для шага с контентом:

```text
Да, всё готово
Частично готов
Нужно подготовить
```

Варианты срочности:

```text
Как можно быстрее
В течение недели
Не тороплюсь
```

Варианты способа связи:

```text
Telegram
WhatsApp
MAX
По телефону
```

Квиз отправляет JSON примерно такого вида:

```json
{
  "name": "Заявка из квиза CTA",
  "contact": "@username или телефон",
  "service": "Квиз на сайте",
  "message": "Ответы из квиза CTA:\nСфера деятельности: ...\nКонтент для сайта: ...\nСрочность: ...\nКонтакт: ...\nУдобнее связаться: ...",
  "_honeypot": "",
  "_page": "/tekuschaya-stranica",
  "quiz": {
    "sphere": "Сфера",
    "content": "Частично готов",
    "urgency": "В течение недели",
    "contact": "@username",
    "contactMethod": "Telegram"
  }
}
```

На стороне PHP-обработчика вложенный объект `quiz` отдельно не используется. Основная информация из квиза попадает в Telegram через поле `message`.

## Серверный обработчик заявок

Основной обработчик:

```text
public/scripts/api/send.php
```

Логика обработчика:

```text
1. Принимает только POST.
2. Читает JSON из тела запроса.
3. Если JSON некорректный, возвращает 400.
4. Если заполнен _honeypot, возвращает ok=true без отправки в Telegram.
5. Нормализует поля name, phone, email, contact, service, message, _page.
6. Валидирует имя, телефон, email, contact, message.
7. Требует хотя бы одно поле связи: phone, email или contact.
8. Загружает Telegram env-переменные из окружения или .env-файлов.
9. Формирует текст заявки.
10. Отправляет сообщение в один или несколько Telegram-чатов.
11. При успешной отправке возвращает {"ok": true}.
```

Env-переменные Telegram:

```text
TELEGRAM_BOT_TOKEN
TELEGRAM_CHAT_IDS
TELEGRAM_CHAT_ID
TELEGRAM_CHAT_ID_2
TELEGRAM_API_BASE
TELEGRAM_TIMEOUT
TELEGRAM_CONNECT_TIMEOUT
TELEGRAM_IPRESOLVE
```

В Telegram уходит:

```text
Имя
Телефон, если заполнен
Email, если заполнен
Контакт из квиза, если заполнен
Услуга, если заполнена
Сообщение, если заполнено
Страница, с которой отправлена заявка
Время отправки в часовом поясе Europe/Moscow
```

Серверный обработчик явно не добавляет в текст заявки:

```text
IP-адрес пользователя
User-Agent
Referer
UTM-метки как отдельные поля
cookies
client id Яндекс.Метрики
fingerprint браузера
```

Но эти данные могут существовать в других местах: в логах веб-сервера/хостинга, в сетевых запросах, в Яндекс.Метрике, у Telegram API, у CDN или прокси, если они используются.

## Старый/альтернативный worker

В проекте также есть Cloudflare Worker:

```text
worker/index.js
```

Он тоже принимает POST, проверяет `_honeypot`, формирует заявку и отправляет ее в Telegram. В текущем фронтенде по умолчанию используется `/scripts/api/send.php`, если не задан `NEXT_PUBLIC_CONTACT_ENDPOINT`.

Особенности worker:

```text
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: POST, OPTIONS
Access-Control-Allow-Headers: Content-Type
```

Worker не добавляет IP/User-Agent в текст заявки, но Cloudflare как платформа может видеть технические параметры запроса.

## Яндекс.Метрика

Метрика подключается глобально в:

```text
src/app/layout.tsx
```

Идентификатор счетчика:

```text
109427452
```

Параметры инициализации:

```text
ssr: true
webvisor: true
clickmap: true
ecommerce: 'dataLayer'
referrer: document.referrer
url: location.href
accurateTrackBounce: true
trackLinks: true
```

Дополнительно компонент:

```text
src/shared/ui/YandexMetrika/YandexMetrika.tsx
```

отправляет `hit` при клиентской навигации между страницами.

Формы заявки после успешного POST вызывают цель:

```text
ym(109427452, 'reachGoal', 'contact_form_submit')
```

Также перед переходом отправляется виртуальный просмотр:

```text
ym(109427452, 'hit', '/thank-you')
```

Это поддерживает URL-цель Метрики `url содержит thank` даже в случае, если пользователь не дождался полной загрузки страницы спасибо.

## Cookie notice

Компонент:

```text
src/shared/ui/CookieNotice/CookieNotice.tsx
```

Показывает пользователю уведомление:

```text
Используем cookie и Яндекс Метрику, чтобы понимать, как пользуются сайтом, и улучшать его.
```

После нажатия кнопки в `localStorage` записывается:

```text
webstudio-cookie-notice-accepted=true
```

Важно: текущее уведомление только скрывает баннер. Оно не управляет фактическим включением/отключением Яндекс.Метрики. Метрика загружается независимо от согласия в баннере.

Есть базовые меры:

```text
Скрытое honeypot-поле _honeypot
Клиентская Zod-валидация
Серверная PHP-валидация
Ограничения длины полей
Проверка формата телефона
Проверка формата email
Разрешен только POST
```

Чего сейчас не видно в коде:

```text
rate limiting
captcha или Turnstile
CSRF-токен
nonce/одноразовый токен формы
проверка Origin/Referer
ограничение CORS для worker
серверная нормализация UTM-меток
разделение аналитики и персональных данных
режим загрузки Метрики только после согласия
маскирование персональных данных в логах
политика хранения и удаления заявок
```

## Потенциальные точки идентификации трафика

Точки, где пользователь или источник трафика может быть идентифицирован технически:

```text
1. Запрос браузера к сайту: IP, User-Agent, TLS/HTTP-параметры, cookies, URL, query string.
2. Запрос POST к /scripts/api/send.php: IP, User-Agent, Origin/Referer, тело заявки, путь страницы.
3. Логи хостинга или веб-сервера: могут содержать IP, URL, User-Agent, Referer, статус ответа.
4. Яндекс.Метрика: счетчик, визиты, страницы, referrer, клики, Вебвизор, карта кликов, цель contact_form_submit.
5. Telegram Bot API: получает текст заявки и техническую информацию о запросе сервера к Telegram.
6. CDN/proxy/WAF, если используется: может видеть сетевые параметры запросов и поведенческие признаки.
7. Browser storage: localStorage хранит факт принятия cookie notice.
8. Thank-you страница: переход на нее может использоваться как косвенный сигнал конверсии в аналитике.
```

Данные, которые прямо попадают в лид:

```text
Имя
Телефон
Email
Контакт в мессенджере или телефон из квиза
Тип услуги
Ответы квиза
Сообщение
Страница отправки
Время отправки
```

Данные, которые прямо не попадают в лид, но могут быть доступны инфраструктуре:

```text
IP
User-Agent
Referer
UTM в URL
cookies
идентификаторы аналитики
поведение на странице
клики и записи Вебвизора
```

- какие изменения внести в PHP-обработчик;
- что настроить в Яндекс.Метрике;
