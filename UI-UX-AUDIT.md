## Проблемы и план исправления

### 1. Аватар директора — только буква "И"

**Файл:** `src/widgets/DirectorMessage/ui/DirectorMessage.tsx:34`

**Проблема:** Блок личного обращения от основателя использует placeholder — одну букву "И" вместо фотографии. Личное обращение без лица не вызывает доверия и снижает конверсию.

**Исправление:**

1. Добавить фото основателя в `public/images/` (рекомендуемый размер: 80×80 или 160×160 px, формат `.webp`)
2. Заменить `<div className={styles.avatar}>И</div>` на компонент `<Image>`:
   ```tsx
   <Image
     src="/images/ivan-avatar.webp"
     alt="Иван, руководитель Project 42"
     width={56}
     height={56}
     className={styles.avatar}
   />
   ```
3. Скорректировать стили `.avatar` — убрать текстовое выравнивание, добавить `border-radius: 50%`

---

### 2. CTA-кнопка скрыта в мобильном меню

**Файл:** `src/widgets/Header/ui/Header.module.scss:86-90`

**Проблема:** Кнопка "Обсудить проект" скрыта через `display: none` на мобильных. В открытом бургер-меню нет ни одного конверсионного действия — пользователь видит только навигационные ссылки.

**Исправление:**

1. В `Header.module.scss` убрать `display: none` у `.cta` для мобильных, вместо этого показывать кнопку внутри открытого меню `.navOpen`
2. Добавить кнопку отдельно в мобильный блок навигации:
   ```tsx
   {
     menuOpen && (
       <Button
         size="md"
         onClick={() => {
           setModalOpen(true)
           setMenuOpen(false)
         }}
         className={styles.mobileCtaInMenu}
       >
         Обсудить проект
       </Button>
     )
   }
   ```
3. Разместить кнопку в конце списка ссылок в открытом меню

### 3. Низкий контраст `$color-text-muted`

**Файл:** `src/shared/styles/variables.scss:7`

**Проблема:** `$color-gray-dark: #6060a0` на фоне `#080812` даёт контраст ~2.8:1. WCAG AA требует минимум 4.5:1 для обычного текста и 3:1 для крупного. Muted-текст используется повсеместно — описания карточек, лейблы, примечания к ценам.

**Исправление:**

1. Заменить `$color-gray-dark: #6060a0` → `#8888bb` (контраст ~4.6:1 на `#080812`)
2. Проверить визуально все места использования `$color-text-muted` после замены
3. Опционально: для самого мелкого текста (`font-size-xs`, `font-size-sm`) поднять до `#9999cc`

### 4. Иконки-кресты в "Что мы не делаем"

**Файл:** `src/widgets/WhatWeDontDo/ui/WhatWeDontDo.tsx:46-52`

**Проблема:** SVG-иконка "×" (крест) ассоциируется с ошибкой, запретом, негативом. Но содержимое пунктов — это позитивные обещания ("не даём ложных обещаний", "прозрачное ценообразование"). Возникает когнитивный диссонанс: глаз видит "запрещено", текст говорит "мы честные".

**Исправление:**

1. Переименовать секцию: `"Что мы не делаем"` → `"Наши принципы"` (eyebrow оставить `"Честность"`)
2. Переформулировать пункты в позитивном ключе:
   - "Не даём ложных обещаний по срокам" → "Честные сроки: обсуждаем все нюансы заранее"
   - и т.д.
3. Заменить X-иконку на checkmark (уже используется в `Pricing.tsx`):
   ```tsx
   <Icon name="check" size={20} className={styles.icon} />
   ```

### 5. Неоптимальный порядок секций

**Файл:** `src/app/page.tsx:62-75`

**Проблема:** `WhatWeDontDo` стоит до социальных доказательств (`Portfolio`, `Stats`, `Testimonials`). `DirectorMessage` оторван от `Testimonials` блоком `Pricing`. Нарушается логика убеждения: сначала нужно показать результаты, потом — принципы.

**Текущий порядок:**

```
Hero → WhyUs → Services → HowItWorks → WhatWeDontDo → Portfolio → Stats → Testimonials → DirectorMessage → Pricing → Cta → Faq
```

**Рекомендуемый порядок:**

```
Hero → WhyUs → Services → HowItWorks → Portfolio → Stats → Testimonials → DirectorMessage → WhatWeDontDo → Pricing → Cta → Faq
```

**Логика:** Сначала показываем что делаем и как → потом доказываем делами (портфолио, цифры, отзывы) → потом личное обращение от основателя → потом принципы → цены → CTA.

### 6. Карточки портфолио некликабельны

**Файл:** `src/widgets/Portfolio/ui/Portfolio.tsx:16-40`

**Проблема:** Карточки проектов не интерактивны — нет ссылки, нет hover-состояния (кроме базового из `card-hover`). Пользователь не понимает, можно ли кликнуть, чтобы узнать подробнее.

**Исправление:**

1. Добавить поле `url` в модель `Project` (если ещё нет)
2. Обернуть карточку в `<Link href={project.url}>` или сделать overlay-ссылку
3. Добавить явный hover: стрелочку / оверлей / текст "Смотреть проект" появляющийся при наведении

```tsx
<li key={project.id} className={styles.card}>
  <Link href={project.url ?? '/portfolio'} className={styles.cardLink}>
    {/* existing content */}
    <span className={styles.hoverOverlay}>Смотреть проект →</span>
  </Link>
</li>
```

### 7. Footer без соцсетей и правовых ссылок

**Файл:** `src/widgets/Footer/ui/Footer.tsx`

**Проблема:** Footer содержит только логотип, навигацию9. Стат "0 брошенных проектов" и кнопку. Нет ссылок на мессенджеры/соцсети, нет политики конфиденциальности. При сборе данных через форму (email, телефон) политика конфиденциальности юридически обязательна (ФЗ-152).

**Исправление:**

1. Добавить иконки/ссылки на Telegram, ВКонтакте и другие актуальные каналы
2. Создать страницу `/privacy` с политикой конфиденциальности
3. Добавить ссылку в нижнюю строку footer:
   ```tsx
   <Link href="/privacy" className={styles.legalLink}>
     Политика конфиденциальности
   </Link>
   ```
4. Опционально: добавить ссылку на email

### 8. Базовый шрифт 14px

**Файл:** `src/shared/styles/variables.scss:42`

**Проблема:** `$font-size-base: 14px` — ниже современного стандарта 16px. Длинные тексты (принципы в `WhatWeDontDo`, параграфы в `DirectorMessage`) читаются с напряжением, особенно на маленьких экранах.

**Исправление:**

1. `$font-size-base: 14px` → `$font-size-base: 16px`
2. Соответственно сдвинуть шкалу:
   - `$font-size-sm: 13px` → `14px`
   - `$font-size-xs: 12px` → оставить
3. После изменения проверить все секции визуально — возможны смещения в карточках и grid-лейаутах

### 9. Стат "0 брошенных проектов"

**Файл:** `src/widgets/Stats/ui/Stats.tsx:19`

**Проблема:** Значение `0` выглядит невыразительно на фоне `100%`, `×3`, `≤4`. Анимация CountUp для нуля не запускается (`static: true`). Пункт читается как "мы ничего не бросили" — слабый аргумент.

**Варианты замены:**

- `∞ — поддержка после сдачи` (подчёркивает долгосрочность)
- `5★ — средняя оценка проектов`
- Убрать этот стат, добавить более конкретный (например, среднее время разработки)

### 10. Нет активного пункта в навигации

**Файл:** `src/widgets/Header/ui/Header.tsx:38-59`

**Проблема:** При скролле по странице ни один пункт меню не подсвечивается как активный. На длинном лендинге это дезориентирует — непонятно, в какой части страницы находишься.

**Исправление:**

1. Добавить `useEffect` с `IntersectionObserver` для отслеживания активной секции
2. Хранить `activeSection` в state
3. Добавить класс `styles.navLinkActive` к совпадающему `href`:
   ```tsx
   <a
     href={link.href}
     className={[styles.navLink, activeSection === link.href.slice(1) && styles.navLinkActive].filter(Boolean).join(' ')}
   >
   ```
4. В `Header.module.scss` добавить стиль:
   ```scss
   .navLinkActive {
     color: $color-text;
   }
   ```
