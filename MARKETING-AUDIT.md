## Топ-5 критических маркетинговых проблем

1. **Hero не отвечает на вопрос «что я получу?»** — заголовок описывает студию, а не выгоду клиента
2. **Единственная точка конверсии** — CTA только в конце, ни одного промежуточного оффера
3. **Социальное доказательство слабое** — 3 клиента без фото, без логотипов, без конкретных метрик
4. **Скарситет не используется как инструмент продаж** — «≤4 проекта» упоминается в WhyUs и исчезает
5. **Директор без фото** — отсутствие лица критично для персонального доверия

## 2. Целевая аудитория — отсутствие ICP

### Что показывает портфолио

| Клиент       | Сегмент                               |
| ------------ | ------------------------------------- |
| Rosa         | Строительная компания (B2B/B2C)       |
| Аудио-мастер | Частный специалист (B2C micro-бизнес) |
| TrueTell     | B2B аналитическая компания            |

Три разных сегмента с разными болями, бюджетами и ЛПРами.

### Проблема

Нет ни одного сигнала на сайте, кому конкретно предназначена студия. Это создаёт проблему:

- Малый бизнес смотрит на цену 150k+ и думает: «дорого, не для меня»
- Enterprise смотрит на портфолио из 3 проектов и думает: «маловато»
- B2B клиент видит «частный мастер по аудиотехнике» и сомневается в позиционировании

### Рекомендация

**Выбрать ICP и обозначить его явно.** Если целевой клиент — малый и средний B2B (что подтверждается ценами и кейсами Rosa и TrueTell), то это нужно показать:

```
Eyebrow к Hero: "Для малого и среднего бизнеса"
```

Или добавить в Services секцию «Для кого»: стартапы, сервисные компании, B2B.

## 3. Воронка и Customer Journey

### Текущая структура страницы

```
Hero → Services → WhyUs → HowItWorks → WhatWeDontDo → Portfolio
→ Stats → Testimonials → DirectorMessage → Pricing → FAQ → CTA
```

### Проблема: единственная точка конверсии

CTA (форма) появляется **только в самом конце**. Кнопка «Обсудить проект» в Hero скроллит страницу вниз до CTA-секции — это 11 блоков прокрутки.

Пользователь, который заинтересовался после WhyUs или Portfolio, должен либо прокрутить страницу до конца, либо уйти. **Горячие лиды теряются в середине воронки.**

### Рекомендация: добавить mid-funnel точки конверсии

| Место                           | Оффер                                       |
| ------------------------------- | ------------------------------------------- |
| После WhyUs                     | Кнопка «Оставить заявку» (в модальное окно) |
| После Portfolio                 | «Хотите такой же?» → форма                  |
| После Pricing                   | Кнопка под каждым тарифом уже ведёт → OK    |
| Sticky header / floating кнопка | «Обсудить проект» в хедере при скролле вниз |

### Проблема: нет низкопорогового оффера

Единственный CTA — «Обсудить проект» (полная сделка). Нет ни одного **промежуточного** предложения для нерешительных клиентов:

- Бесплатный аудит текущего сайта
- Шаблон брифа для скачивания
- «Получить пример ТЗ»

## 4. CTA — анализ эффективности

### Текущие CTA

| CTA                 | Место         | Оценка | Проблема                             |
| ------------------- | ------------- | ------ | ------------------------------------ |
| «Обсудить проект»   | Hero          | 6/10   | Generic, нет выгоды в тексте кнопки  |
| «Посмотреть работы» | Hero (ghost)  | 7/10   | Хорошо, альтернативный путь          |
| «Обсудить проект»   | Pricing cards | 6/10   | Одинаковый текст у всех трёх тарифов |
| «Обсудить задачу»   | Custom план   | 7/10   | Хорошо отличается                    |
| CTA секция в конце  | Bottom        | 5/10   | Слабый заголовок, нет срочности      |

### CTA-секция — детальный анализ

```
Текущее:
"Давайте создадим ваш следующий проект"
"Расскажите о задаче — обсудим, как её решить"
```

**Проблема 1:** «Следующий проект» предполагает, что клиент уже делал сайты. Для первого раза — отчуждающая формулировка.

**Проблема 2:** Нет использования скарситета. WhyUs говорит «Не более 4 проектов в работе» — это мощный триггер дефицита. В CTA он не используется совсем.

**Рекомендация:**

```
Вариант с дефицитом:
Heading: "Сейчас мы берём 1–2 новых проекта"
Sub: "Расскажите о задаче — решим, подходим ли мы друг другу"
CTA: "Забронировать место" / "Обсудить проект"
```

### ContactForm — анализ полей

Поля: Имя, Телефон, Тип проекта, Сообщение.

**Проблемы:**

- **Нет email** — звонок как единственный способ связи отпугивает часть аудитории (B2B часто предпочитает переписку)
- **«Сообщение»** — пустой textarea без подсказки снижает конверсию; лучше placeholder: «Какую задачу нужно решить? (Например: переделать сайт, запустить рекламу, оба)»
- **Успех «в течение 24 часов»** — вполне нормально, но можно добавить: «обычно отвечаем в тот же день»

## 5. Социальное доказательство

### Текущее состояние

| Тип                     | Есть | Качество | Проблемы                               |
| ----------------------- | ---- | -------- | -------------------------------------- |
| Текстовые отзывы        | 3    | Medium   | Нет фото, только инициалы              |
| Логотипы клиентов       | Нет  | —        | Потеря credibility                     |
| Цифровые результаты     | Нет  | —        | ROI упомянут в отзыве, но не выделен   |
| Фото директора          | Нет  | —        | Только буква «И»                       |
| Ссылки на живые проекты | Есть | Medium   | audio-repair.vercel.app = vercel домен |
| Кейсы с метриками       | Нет  | —        | Не описан рост трафика, конверсии, ROI |

### Критические проблемы

**1. «Частный мастер по ремонту аудиотехники» в портфолио подрывает B2B позиционирование.** Если студия позиционируется как партнёр для бизнеса с бюджетами 150k+, этот клиент — антипример. Потенциальный клиент (средний B2B) смотрит на него и думает «они работают для микробизнеса».

**2. audio-repair.vercel.app** — продакшн-сайт на бесплатном Vercel-поддомене. Это разрушает доверие: студия продаёт за 150k+, а собственный клиент сидит на vercel.app? Если это демо — надо пометить явно или убрать.

**3. Отзыв t1 (Олег К.) содержит золотой факт:** «окупились менее чем через 2 месяца» — это лучший маркетинговый аргумент на всём сайте, но он похоронен в длинном тексте отзыва. Это должно быть в Stats или выделено визуально.

**4. Нет фото директора.** DirectorMessage — потенциально самая мощная секция для доверия (личный контакт, человек за брендом). Буква «И» вместо фото почти нивелирует её эффект.

### Рекомендации

- Добавить фото Ивана в DirectorMessage (даже неформальное)
- Добавить логотипы Rosa и TrueTell (можно не весь логотип — просто название крупно)
- Перенести «окупились менее чем через 2 месяца» в Stats как отдельную карточку или цитату
- Для audio-repair: либо прописать «учебный проект» / убрать, либо получить нормальный домен для клиента
- Добавить в каждый проект портфолио раздел «Результат»: цифры трафика, конверсии, сроки

## 6. Месседжинг и контент

### Проблема «отрицательного» позиционирования

**WhatWeDontDo** — весь раздел построен на отрицаниях («Не даём», «Не навешываем», «Не ставим», «Не накручиваем»). Проблема: люди хуже запоминают негативные конструкции.

Дополнительная проблема: раздел идёт **после HowItWorks**, когда клиент уже понял процесс. Негативный месседж в этом месте сбивает позитивный momentum.

**Решение:** Переформулировать в позитив:

| Текущее (негативное)                           | Замена (позитивное)                                    |
| ---------------------------------------------- | ------------------------------------------------------ |
| «Не даём ложных обещаний по срокам»            | «Фиксируем дедлайн в договоре и соблюдаем его»         |
| «Не навешываем дополнительные ненужные услуги» | «Предлагаем только то, что реально работает на задачу» |
| «Не ставим дизайн в противовес эффективности»  | «Красиво и конвертирует — не выбирайте одно»           |
| «Не накручиваем поведенческие факторы»         | «Только white-hat SEO: долго, но надёжно»              |

Или вообще **убрать раздел** — эти мысли уже сказаны лучше в WhyUs.

### Технический слэнг для не-технической аудитории

В FAQ и описаниях есть: «Next.js», «headless CMS», «Sanity, Strapi, Contentful», «TypeScript». Если клиент — малый B2B (директор компании, не CTO), это создаёт барьер.

Рекомендация: в публичных текстах (FAQ, Services) — только результат, не технологии.

```
Текущее: "Основной стек — Next.js, React, TypeScript"
Замена: "Быстро грузится, легко обновлять, не привязан к одному разработчику"
```

### Проблема нарратива «без посредников» vs «Выделенный менеджер»

В WhyUs: «Менеджеры съедают бюджет → Только работа и результат»  
В Pricing (тариф «Индивидуально»): «Выделенный менеджер проекта»

**Это прямое противоречие.** Клиент, дочитавший до Pricing, увидит несоответствие.

Рекомендация: заменить «Выделенный менеджер» на «Прямой контакт с командой» или «Личный куратор проекта».

## 7. Психология ценообразования

### Что работает

- Три тарифа с якорением на средний — классика и правильно
- «Популярный выбор» badge — снижает паралич выбора
- «от» в цене — честно сигнализирует о минимальном пороге
- Порядок фич совпадает с приоритетами клиента

### Что не работает

**«Для старта и проверки гипотез»** как note к лендингу звучит как «бюджетный вариант». Клиент, у которого есть задача (а не «гипотеза»), может почувствовать, что этот тариф — не для серьёзных.

Лучше: «Для запуска и быстрого результата»

**Нет информации об условиях оплаты.** Для B2B важно: предоплата? рассрочка? поэтапная оплата? Отсутствие этой информации создаёт неопределённость, которая тормозит решение.

**Нет гарантии или снижения риска.** Ни на одной из страниц нет риверсала риска:

- «Если не понравился дизайн на этапе прототипа — вернём предоплату»
- «Фиксированный дедлайн в договоре»
- Это уже есть в FAQ, но не в Pricing рядом с ценой

## 8. Конкурентная дифференциация (WhyUs)

### Оценка: 8/10 — сильнейший раздел сайта

VS-таблица — нестандартный и эффективный формат. Конкретные противопоставления создают дифференциацию без прямого упоминания конкурентов.

### Что можно усилить

**Добавить доказательства к каждому пункту.** Сейчас это утверждения без proof:

| Утверждение «Мы»               | Доказательство (добавить)                           |
| ------------------------------ | --------------------------------------------------- |
| «×2 быстрее за счёт AI»        | Ссылка на Stats секцию или цифра в карточке         |
| «Не более 4 проектов в работе» | «Сейчас свободно 2 места» (если актуально)          |
| «Честная цена»                 | «Смета — это договор, не начало переговоров»        |
| «Чистый код без замков»        | «Вы получаете исходный код и доступы с первого дня» |

## 9. Геотаргетинг

Сайт — без единого упоминания города или региона. Для локального SEO и доверия российского B2B-клиента это проблема:

- «Веб-студия Москва» генерирует в разы больше целевых запросов, чем просто «веб-студия»
- Российский B2B предпочитает работать с «понятными» по географии партнёрами

Рекомендация: добавить город в footer, в meta description и в hero, если есть геопривязка.

## 10. Тон и стиль

### Хорошо

- DirectorMessage написано живо, по-человечески — сильнейший текст на сайте по тону
- HowItWorks конкретен и структурирован
- FAQ отвечает на реальные возражения

### Можно улучшить

- **Избыточное «мы»:** почти все секции начинаются с «мы» / «наш». Клиент хочет читать про себя, не про студию.
- **«Остаёмся рядом столько, сколько нужно»** повторяется дважды (HowItWorks шаг 6 и Services → Поддержка). Лучше разнообразить.
